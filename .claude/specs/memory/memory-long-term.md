## Storage Strategy

### Semantic & Procedural (Redis)
- Stored as JSON-serialized Pydantic models
- Key: `memory:semantic:{user_id}` and `memory:procedural:{user_id}`
- No TTL (permanent)
- Fast O(1) reads

```python
# Behind the scenes
redis_key = f"memory:semantic:{user_id}"
profile_json = profile.model_dump_json()
redis.set(redis_key, profile_json)  # No TTL
```

### Episodic (ChromaDB)
- Stored as documents with metadata
- Collection: `episodic_{user_id}`
- Embeddings auto-generated by ChromaDB
- Searchable by similarity

```python
# Behind the scenes
collection = client.get_or_create_collection(f"episodic_{user_id}")
collection.add(
    ids=[doc.doc_id],
    documents=[doc.document],
    metadatas=[{
        "type": doc.type,
        "importance": doc.importance,
        "tags": doc.tags,
        "created_at": doc.created_at.isoformat()
    }],
    embeddings=None  # ChromaDB auto-embeds
)
```

## Retrieval Patterns

### Load User Memory for Chat
```python
# Typical flow
semantic = await load_semantic(user_id)
procedural = await load_procedural(user_id)
episodic = await search_episodic(user_id, current_query, limit=5)

# Returns LongTermMemory object
ltm = LongTermMemory(
    semantic=semantic,
    procedural=procedural,
    episodic_chunks=[doc.document for doc in episodic]
)
```

### Update After Summarizer
```python
# Summariser extracts facts/prefs
updates_sem = {"income_monthly": 120000, "financial_goals": [...]}
updates_proc = {"tone": "formal"}

await update_semantic(user_id, updates_sem)
await update_procedural(user_id, updates_proc)
```

### Monitor Events to Episodic
```python
# Monitor detects commodity alert
event = MonitorEvent(
    event_type="commodity_alert",
    severity="alert",
    message="Wheat price surge",
    commodities=["wheat"]
)

doc = event.to_episodic_doc(user_id, session_id)
await save_episodic(doc)

# Also update semantic
await update_semantic(user_id, {
    "commodity_interests": [..., "wheat"],
    "last_monitor_alert": datetime.now().isoformat()
})
```

## Search Behavior

**search_episodic()** returns documents ranked by:
1. Cosine similarity (embedding-based)
2. Importance score (0.0-1.0)
3. Recency (most recent first)

Filter by `min_importance` (default 0.3) to exclude low-priority memories.

Example:
```python
# Get top 5 most relevant episodic memories for a query
docs = await search_episodic(
    user_id="user_123",
    query="How much should I save for retirement?",
    limit=5,
    min_importance=0.3
)
# Returns docs ranked by relevance + importance
```

## Data Retention

- **Semantic**: Permanent (never deleted except on user request)
- **Procedural**: Permanent (updated periodically)
- **Episodic**: Permanent (but can be archived after 1 year)
- **Sessions**: 2 hours (TTL_SESSION)

## Usage Example

```python
from memory.long_term import (
    save_semantic, load_semantic,
    save_episodic, search_episodic
)
from memory.models import SemanticProfile, EpisodicDoc

# Save user facts
profile = SemanticProfile(
    name="Ashish",
    income_monthly=100000,
    financial_goals=["retirement", "home"]
)
await save_semantic("user_123", profile)

# Later: retrieve for chat
loaded = await load_semantic("user_123")
print(loaded.financial_goals)  # ["retirement", "home"]

# Search for context
query = "Tell me about my goals"
relevant = await search_episodic("user_123", query, limit=5)
for doc in relevant:
    print(doc.document, f"(importance: {doc.importance})")
```
